更新时间：2026-06-26

角色：美术 / UX

状态：review

## 结论

第一批 12 项够开发开工。

原因很简单：它覆盖了“看订单、点商品、热食处理、READY 交付、收银反馈”的最小闭环。开发不需要等全量美术、顾客重画、背景重画或 HUD 精修。

开工条件：

- 订单气泡、READY、商品卡、微波炉四组资源先定版。
- 文字、数字、进度条、错误反馈先由程序节点承担。
- `reference_slice_*` 和 `placeholder_product_card_*` 只做参考 / QA 占位，不作为正式资源。

## 390x844 最容易失败的 5 个视觉风险

1. READY 胶囊太小或被订单内容挤压，玩家看不到交付入口。
2. 订单槽低于 44px，商品图标和数量在多品订单里糊成一团。
3. 商品卡库存数字被 needed glow、热食角标或售罄遮罩挡住。
4. 微波炉 heating / ready 只靠小字区分，窄屏下玩家不知道能不能取。
5. 当前顾客气泡和等待顾客气泡层级不够，玩家误点等待订单。

## 只能做 4 项时的顺序

1. 订单气泡底板：当前 / 等待两张一起做，否则层级不成立。
2. READY 胶囊：交付入口必须明确。
3. 商品卡底板 + 热食角标：商品点击来源和热食判断一起成立。
4. 微波炉三态：热食中间流程必须可读。

收银金币、呼吸光、错误红闪、售罄遮罩先交给程序，不挡开发开工。
