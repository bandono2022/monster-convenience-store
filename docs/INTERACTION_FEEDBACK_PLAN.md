# 交互反馈优化计划

## 目标

玩家每次点击后，应在 0.2 秒内明确知道：

- 点击是否生效。
- 商品是否交付成功。
- 商品是否需要加工。
- 加工进行到了哪一步。
- 下一步应该点击哪里。

## 第一步：基础状态反馈

状态：已完成

- 当前服务顾客使用醒目的边框和“正在服务”标签。
- 当前订单需要的商品显示“订单需要”或“需加热”。
- 商品库存标签始终可见。
- 正确、错误、补货和加热使用不同颜色的底部反馈。

## 第二步：加工流程反馈

状态：已完成

- 热饭团订单依次显示“需加热、加热中、可取、已交付”。
- 微波炉显示剩余时间、进度条和所属顾客。
- 加热完成后微波炉切换为高亮状态，并提示点击交付。
- 切错顾客时阻止交付，并提示正确顾客位置。

## 第三步：操作动效与声音

状态：下一阶段

- 商品点击时增加按下、回弹和飞向顾客的动画。
- 错误商品增加短促抖动和红色叉号。
- 加热完成时增加微波炉弹跳、闪光和提示音。
- 订单完成时增加金币飞行、顾客表情变化和连击动效。
- 为点击、交付、错误、加热完成和订单完成配置短音效。

## 第四步：新手引导与难度衔接

状态：下一阶段

- 第一次订单使用手势提示引导选择顾客和商品。
- 第一次热饭团订单分步骤引导放入、等待和取出。
- 前期营业日显示“订单需要”高亮，后期逐步减少辅助。
- 高难度保持调度压力，但不依赖看不清或不知道点击结果。

## 第五步：测试与验收

状态：持续进行

- 使用手机竖屏比例验证所有标签可读且不遮挡。
- 连续测试正确交付、错误商品、库存耗尽、补货、加热和切错顾客。
- 邀请首次接触游戏的玩家试玩，观察是否会在 30 秒内独立完成热饭团订单。
- 记录误触次数、首单完成时间和首次加热流程完成率。
