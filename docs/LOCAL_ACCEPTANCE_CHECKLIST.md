# 本地商业化验收清单

更新时间：2026-06-26

用途：本地手工验收经营页是否达到“能试玩、能判断下一步商业化缺口”。每次验收直接勾选，不需要写长报告。

## 验收环境

- [ ] Cocos Creator 3.8.8 打开项目。
- [ ] 运行场景：`assets/scenes/Game.scene`。
- [ ] 手机尺寸至少测一次：390x844。
- [ ] 验收前记录当前分支/本地变更状态：`git status --short`。

## 第一次打开

- [ ] 第一次进入经营中心，玩家能看懂资金、口碑、店铺等级和开始营业入口。
- [ ] 第一次进入营业页，玩家能在 5 秒内看出：当前顾客、订单气泡、商品卡、微波炉、收银台。
- [ ] 不读文档也能尝试点击订单需要的普通商品。
- [ ] 首轮提示不遮挡主要点击区。

## D-004 经营页手动点测

### 普通商品与订单气泡

- [ ] 点击当前顾客订单里的普通商品后，商品进入当前顾客订单气泡。
- [ ] 普通商品点击后不直接收钱、不直接完成订单。
- [ ] 多商品订单第一件完成后显示 partial 状态。
- [ ] 多商品订单备齐后订单气泡显示明显 READY 状态。
- [ ] READY 订单气泡可点击交付。
- [ ] 当前顾客已 READY 时，点击当前顾客也可完成交付。

### 未 READY 与错误路径

- [ ] 点击未 READY 订单气泡不会交付。
- [ ] 点击未 READY 当前顾客不会交付。
- [ ] 未 READY 时提示缺失商品名或“订单气泡还缺”。
- [ ] 错误商品会提示错误，顾客有负面反馈，订单气泡/商品有错误反馈。
- [ ] 售罄商品会提示售罄，并按当前订单是否需要该商品处理。
- [ ] 补货中商品会提示正在补货，不误完成订单。

### 热食与微波炉

- [ ] 热食商品点击后进入微波炉，不直接进入订单气泡。
- [ ] 微波炉加热中显示进度/加热反馈。
- [ ] 微波炉完成后有 ready/完成反馈。
- [ ] 点击完成的微波炉后，热食回到所属顾客订单气泡。
- [ ] 热食属于 A 顾客时，切到 B 顾客点击微波炉不会混给 B 顾客。
- [ ] A 顾客热食回填后，A 顾客订单气泡可继续 partial 或 READY。

### 交付、收银、HUD

- [ ] READY 订单气泡交付后顾客变开心或进入完成反馈。
- [ ] 收银台有入账/闪烁/金币反馈。
- [ ] HUD 主数值显示本轮营业额 `currentShiftRevenue`，成功交付后增加。
- [ ] 营业中不把 `wallet` 即时增加作为入账通过标准。
- [ ] 连击、收入、小费反馈不遮挡下一次核心点击。
- [ ] 完成顾客离场/换客后，下一单仍可继续操作。

### 托盘降级

- [ ] 中间备餐台不再显示 READY。
- [ ] 中间备餐台不再显示商品堆放。
- [ ] 玩家不需要点击备餐台完成订单。

## 390x844 手机尺寸

- [ ] HUD 不遮挡暂停按钮或主要内容。
- [ ] 当前订单气泡完整可读、可点击。
- [ ] 顾客脸不被订单气泡或柜台严重遮挡。
- [ ] 微波炉可看出 idle/heating/ready 状态并可点击。
- [ ] 商品卡可点击，商品名/库存不互相遮挡。
- [ ] Toast/反馈条不长期遮挡订单气泡、微波炉、商品卡。

## 核心冒烟路径

### Day 1

- [ ] 开始营业后能完成第一单普通商品。
- [ ] 第二单仍能按订单气泡 READY 交付。
- [ ] 90 秒营业过程中无卡死、无明显点击失效。
- [ ] 结算页能显示收入/利润/评价。

### Day 3

- [ ] Day 3 进入营业后，能看出商品、库存、顾客压力有变化。
- [ ] 至少完成一单多商品订单，并看到 partial 到 READY。
- [ ] 出现缺货/补货时反馈清楚，不误收钱。
- [ ] 结束后能回到经营中心继续升级或采购。

### Day 7

- [ ] Day 7 进入营业后，核心 HUD 和订单气泡仍可读。
- [ ] 至少完成一单含热食或设备相关订单。
- [ ] 多顾客切换时，准备中的订单不会混给另一位顾客。
- [ ] 结束后数值反馈能支持判断下一步平衡问题。
