# M2 经营循环计划与验收

更新时间：2026-06-17

## 目标

M2 的目标是让玩家可以循环进行：

经营中心 -> 选择策略 -> 经营前准备/进货 -> 90 秒营业 -> 结算 -> 升级/解锁 -> 选择下一轮策略 -> 返回经营中心 -> 下一轮。

## 当前已具备

| 模块 | 状态 | 说明 |
| --- | --- | --- |
| 经营中心 | 已有 | 展示轮次、资金、口碑、热门品类、策略、库存和设备等级 |
| 策略选择 | 已有 | 非试营业轮需要选择策略，策略影响下一轮营业 |
| 经营前准备 | 已有 | 展示热门品类、资金、仓库容量、商品库存和采购按钮 |
| 自动补货建议 | 已有 | `补至基础库存` 可按资金/容量补到目标库存 |
| 采购确认 | 已有 | 校验资金、仓库容量、可用库存后进入营业 |
| 结算页 | 已有 | 展示营业额、成本、净利润、资金、星级、完成/流失 |
| 升级/解锁 | 已有 | 结算页支持解锁商品、升级货架/微波炉/店铺 |
| 下一轮推进 | 已有 | 选择策略后返回经营中心，进入下一轮准备 |
| 准备页返回 | 已补 | 经营前准备页可以返回经营中心，采购清单清空 |

## 当前主要缺口

1. 经营中心还是功能面板，不是“便利店外景首页”。
   - 这是 M3 美术阶段更适合处理的内容。
   - M2 只要求流程可玩，不要求正式首页场景。

2. 升级页还嵌在结算页。
   - 目前可玩，但信息密度偏高。
   - 后续应拆成独立升级页面，结算页只做复盘和入口。

3. 进货建议还偏轻。
   - 现在有库存、上轮销量、热门品类、短缺/正常/积压。
   - 后续可以加入“推荐采购量”，但不应给精确答案，避免变成自动填表。

4. 下一轮策略入口有两处。
   - 结算页可以选，经营中心没有策略时也可以选。
   - 当前能避免流程死锁，后续 UI 美化时再统一。

## M2 最小验收脚本

1. 新账号进入游戏，看到经营中心。
2. 点击经营前准备，能进入进货页。
3. 在进货页点击返回经营中心，能返回且不会开始营业。
4. 再进入进货页，点击补至基础库存，采购数量和支出变化。
5. 点击确认采购并营业，进入 90 秒营业。
6. 营业结束后进入结算页。
7. 在结算页至少执行一个升级/解锁，资金扣除并显示反馈。
8. 在结算页选择下一轮策略。
9. 点击返回经营中心，轮次推进到下一轮。
10. 从经营中心再次进入经营前准备并开始下一轮营业。

## 下一步建议

优先做低成本流程完善：

1. 给结算页增加“下一步”层级：先复盘，再升级/策略。
2. 给经营中心增加更明确的“库存风险”提示。
3. 给进货页增加推荐采购量，但保留玩家决策。
4. 再拆独立升级页。

暂时不要做：

- 外景首页正式美术。
- 装修系统。
- 多店铺/多设备复杂经营。
- 精确需求预测。
