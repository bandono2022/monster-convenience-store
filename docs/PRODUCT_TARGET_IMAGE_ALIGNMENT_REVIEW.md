## 目的

这份文档只从产品和玩法角度审查当前经营页目标图：

- 目标图是否服务当前经营页 V1 玩法。

结论：

## 建议改目标图的地方

### 1. 工作台改成“单托盘 + 微波炉 + 收银台”

当前目标图中间区域像传送带，会让玩家以为可以提前备餐或多物品排队。

目标图 V2 应改为：

- 左侧：微波炉，一个处理位。
- 中间：托盘，一个当前订单组装位。
- 右侧：收银台或付款反馈位。
- 托盘上可以显示 1 至 3 个商品槽，但它们都属于同一个顾客订单。
- 不出现像传送带一样的多成品队列。

玩法不改。

### 2. `DELIVERED` 改为瞬时反馈，不做独立操作面板

当前目标图右侧大 `DELIVERED` 面板很醒目，但它会暗示玩家要点右侧交付。

目标图 V2 应改为：

- READY 时托盘本身发光并成为点击目标。
- 点击托盘后，订单气泡或收银台短暂出现 `DONE` / 勾选 / +收入。
- 右侧保留收银台，不做第二个交付按钮。

玩法不改。

### 3. 微波炉不要表现成多成品缓存

当前目标图里微波炉内有饭团，下方又出现饭团，容易像“已经有多个热食在排队”。

目标图 V2 应改为：

- 加热中：商品在微波炉内，显示进度。
- READY：商品仍在微波炉内或出口处，显示 READY。
- 取出后：商品出现在托盘上，微波炉恢复 idle。
- 同一件商品不要同时出现在微波炉和托盘。

玩法不改。

### 4. 商品卡必须显示库存

当前目标图商品卡非常干净，但 V1 的经营压力包含库存、补货、售罄。

目标图 V2 应增加：

- 每张商品卡右下或底部小库存徽章，例如 `x3`。
- 售罄/补货中/未解锁状态要有状态牌。
- 商品图标仍然是最大信息，不让库存抢主视觉。

玩法不改。

### 5. 热食商品先只保留饭团

当前目标图让饭团和布丁都像需要加热。V1 目前只需要一个热食入口即可证明微波炉玩法。

目标图 V2 应改为：

- P0 只有饭团带火焰角标。
- 布丁先作为普通高价商品或后续解锁商品。
- 如果后续要让布丁也加工，放到第 4 天以后或 P2，不进入当前经营页 V1。

玩法暂不扩。

### 6. 商品卡“星星”含义要改清楚

目标图里星星可能表示 needed，也可能表示热门/奖励。这个会直接影响玩家点击判断。

产品建议：

- 当前订单需要的商品：使用强高亮边框或明确 `needed` 角标。
- 非当前顾客需要的商品：P0 不在商品卡上强提示，避免玩家混单。
- 热门/趋势商品：不要和当前订单需要使用同一个星星符号。
- 如果保留星星，建议只表示连击/评价，不表示商品是否需要。

玩法说明需要澄清。

### 7. 底部图例不要常驻

目标图底部图例能解释状态，但商业化经营页不应该长期靠图例。

目标图 V2 应改为：

- 新手第 1 天前几单可以显示轻量图例。
- 常规经营页不常驻大图例。
- 商品卡、订单气泡、托盘状态本身要能表达含义。

玩法不改，只调整教程呈现。

### 8. 顶部第三 HUD 位需要定名

目标图第三个 HUD 是星星 `x3` 和进度条。产品判断它可以保留，但必须定义含义。

建议定义为：

- P0：服务连击 / 当前评价槽。
- 成功订单增加，错误、超时、缺货中断或降低。
- 不在第 1 天前 15 秒制造复杂解释。

这是小幅玩法/UI说明补充，不是新系统。

## 建议修改玩法规格的地方

不是大改，只是补充澄清：

1. 明确目标图中的 `DELIVERED` 是交付成功反馈，不是独立按钮。
2. 明确商品卡必须显示库存，因为库存是 V1 压力来源。
3. 明确 P0 商品卡只强提示当前服务顾客所需商品，不强提示非当前顾客，避免混单。
4. 明确顶部第三 HUD 位使用“服务连击/评价”二选一，不再模糊成星星装饰。
5. 明确 P0 只有一个热食加工商品，后续再扩多热食。

这些修改不改变核心玩法，只让美术和开发不要误读。

竖屏 750x1334。

## 产品结论

旧目标图可以继续作为风格参考，但不能作为最终玩法目标图直接拆资源。

V2 目标图已经确认作为当前目标。

建议下一步：

1. 将 `A-004：经营页目标图 V2 修订` 标记为完成。
2. 解除 `A-002：P0资源清单` 阻塞。
3. 基于 V2 制作全量美术资源与动效需求拆解。

产品最终建议：

```text
以 V2 为目标。
美术资源基本全量重做。
玩法改为订单气泡 READY 交付。
不要恢复独立交付托盘、传送带或多备餐系统。
```
