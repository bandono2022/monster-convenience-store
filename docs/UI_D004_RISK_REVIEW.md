# D-004 UI 风险复查

更新时间：2026-06-26

角色：美术 / UI Hume

状态：review

## 结论

已有文档足够支持开发/测试继续验 D-004；本轮不扩展大规格。

最小风险点只有一个：玩家必须在 390x844 和桌面视口下都能看懂“订单还缺什么、什么时候 READY、热食在哪、交付后有没有收钱”。

## 信息层级风险

| 模块 | 390x844 风险 | 桌面视口风险 | 判定 |
| --- | --- | --- | --- |
| 营业 HUD | 金币、倒计时、暂停/返回挤在顶部，容易压住顾客气泡 | HUD 太分散，金币反馈和收银台距离过远 | HUD 只保留关键数字，反馈不能挡订单 |
| 订单气泡 | 当前气泡、等待气泡、顾客脸容易互相遮挡；槽位小于 44px 会不可读 | 等待气泡太亮会抢当前订单注意力 | 当前气泡必须最高层级，等待气泡降饱和 |
| READY 状态 | READY 胶囊太小会被槽位和数量挤掉，玩家不知道可点 | READY 只靠颜色时，远距离不够明确 | READY 要有独立胶囊 + pulse，点击热区不小于气泡主体 |
| 微波炉热食状态 | heating/ready 若只靠小字，窄屏看不出能否取餐 | 微波炉离商品区太远时，热食流程断开 | idle/heating/ready 必须不用读字也能区分 |
| 收银反馈 | `+金币`、toast、连击反馈可能挡下一次点商品/气泡 | 金币飞行动线太长，反馈弱或干扰视线 | 收银反馈短、轻、靠收银台，不覆盖核心点击区 |

## 最短验收清单

开发/测试只需要确认这些视觉状态“看得懂/点得到”：

- 当前顾客订单气泡一眼比等待顾客更重要。
- 普通商品点完后，订单气泡能看出 partial。
- 订单备齐后，READY 胶囊明显出现，并且点气泡能交付。
- 未 READY 点击气泡时，能看出缺什么或至少知道没交付成功。
- 热食点击后，微波炉进入 heating，完成后明显变 ready。
- 微波炉 ready 点击后，热食回到所属订单气泡。
- 错误商品、售罄、补货中不会像成功状态。
- 收银台有入账反馈，HUD/收入反馈不挡下一次操作。
- 390x844 下订单气泡、微波炉、商品卡三类核心点击区都能点到。
- 桌面视口下等待气泡、HUD、收银反馈不抢当前 READY 气泡注意力。

## P0 视觉通过线

- READY 胶囊：不低于 44px 可点高度，文字或动态 Label 不被数量遮挡。
- 订单槽：多商品订单仍能分清已填/未填/错误。
- 微波炉：idle、heating、ready 三态不用读说明文字也能分辨。
- 商品卡：库存、热食角标、售罄遮罩三者不互相遮挡。
- 收银反馈：只做短反馈，不长时间覆盖订单气泡、商品卡、微波炉。

## 不扩范围

本轮不新增全量美术计划，不要求重画顾客、背景、商品全量 icon 或 HUD 精修。
