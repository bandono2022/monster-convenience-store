# 核心页面信息收缩方案 V1

目标：保留经营深度，减少常驻文字和重复数据，让正式美术能够建立清晰层级。

## 通用原则

- 每个页面只回答一个主要问题。
- 主要操作最多一个，次要操作最多两个。
- 系统逻辑可以复杂，但常驻界面只显示当前决策需要的信息。
- 颜色和图标表达状态，文字只负责名称、数量与结果。
- 历史数据、价格详情和规则说明不常驻显示。

## 经营中心

主要问题：下一轮是否准备好开始？

保留：

- 金币、店铺等级、明日热门。
- 当前经营策略。
- 库存紧缺商品数量。
- 经营前准备按钮。

隐藏或后移：

- 六张完整库存卡片。
- 每件商品的上轮销量。
- 所有设备等级长文本。
- 升级操作，后续放入独立升级页面。

## 经营前准备

主要问题：哪些商品需要补，补多少？

保留：

- 金币、仓库容量、采购支出。
- 商品图片、商品名称、当前库存、采购数量。
- 紧缺、正常、积压的小型状态标记。
- 明日热门标记。
- 加减按钮、基础补货与确认营业。

隐藏或后移：

- 商品卡片上的上轮销量。
- 商品卡片上的进价与售价。
- 经营策略说明。
- 精确需求预测。

## 营业页面

主要问题：下一步应该服务谁、交付什么？

保留：

- 顾客、订单、耐心。
- 当前服务对象的明确边框。
- 商品库存、微波炉状态。
- 营业时间、营业额。

隐藏或弱化：

- 重复的当前顾客文字。
- 大面积收银台信息。
- 长篇操作提示。
- 不影响当前操作的经营数据。

## 营业结算

主要问题：这轮赚得怎么样，下一步做什么？

保留：

- 净利润、评价、缺货次数。
- 一句经营复盘建议。
- 返回经营中心。

隐藏或后移：

- 全量历史统计。
- 所有升级卡片同时展示。
- 三种策略的长说明。
- 商品级详细报表。
