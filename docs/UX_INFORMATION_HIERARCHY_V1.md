更新时间：2026-06-25

角色：美术 / UX

状态：draft

目标图：

`assets/ui/mockups/gameplay-main-order-bubble-ready-v2.png`

## 这份文档解决什么

经营页 V2 的核心不是把所有信息都做大，而是让玩家在 1 秒内判断：

1. 当前服务谁。
2. 这位顾客缺什么。
3. 现在该点商品、等微波炉，还是点 READY 交付。
4. 成功、错误、低耐心和收款反馈从哪里发生。

本版本以“订单气泡 READY 交付”为核心，删除独立交付托盘的视觉优先级。

## 信息分组

### 常驻信息

常驻信息必须稳定可见，不靠弹窗、不靠长说明。

| 信息 | 位置 | 常驻原因 | 显示方式 |
| --- | --- | --- | --- |
| 营业额 / 金币 | HUD 左侧 | 玩家需要确认收入增长 | 图标 + 大数字，弱化文字标签 |
| 剩余时间 | HUD 中心 | 营业压力最高优先级 | 时钟图标 + 大数字 + 进度条 |
| 连击 / 星级 / 满意度 | HUD 右侧 | 反馈经营表现 | 图标 + 数字，允许比时间弱一级 |
| 暂停 | HUD 右上 | 系统控制 | 独立图标按钮 |
| 当前顾客 | 顾客区主焦点 | 决定商品投放归属 | 高亮边框、脚底光、编号或选中态 |
| 当前顾客订单气泡 | 顾客头顶 | 当前操作目标 | 商品槽、缺失/已填/READY 状态 |
| 顾客耐心 | 贴近顾客或订单 | 服务风险 | 竖条或短条，低耐心变色 |
| 商品区 | 底部主操作区 | 玩家点击来源 | 商品图标、库存、热食/需要角标 |
| 微波炉状态 | 工作台左侧 | 热食流程的中间状态 | idle/heating/ready 三态 + 进度 |
| 收银台位置 | 工作台右侧 | 交付成功的结果归因 | 常驻设备，付款时短时激活 |

### 操作反馈

操作反馈跟随玩家动作出现，持续时间短，但必须明确落点。

| 操作 | 反馈落点 | 反馈方式 |
| --- | --- | --- |
| 点击当前顾客 | 当前顾客舞台、订单气泡 | 高亮轻弹，订单气泡描边强化 |
| 点击非当前顾客 | 被点击顾客 | 高亮移动，原顾客高亮淡出 |
| 点击正确普通商品 | 商品卡到当前订单气泡 | 商品小图标飞入对应槽，槽位弹跳 |
| 点击需加热商品 | 商品卡到微波炉 | 商品飞入微波炉，炉内进度启动 |
| 微波炉完成 | 微波炉到订单气泡 | 绿色 ready 灯、蒸汽，取出后填槽 |
| 点击 READY 气泡 | 当前顾客、订单气泡、收银台 | DONE、顾客开心、收银金币联动 |
| 点击未 READY 气泡 | 订单气泡缺失槽 | 红闪、shake、缺失槽提示 |
| 点击错误商品 | 错误商品卡 + 当前订单气泡 | 商品卡红闪，订单气泡轻 shake |

### 短时反馈

短时反馈服务“情绪”和“结果”，不得长期遮挡主操作。

| 反馈 | 建议时长 | 最大遮挡范围 |
| --- | ---: | --- |
| READY 弹出 | 180-240ms 入场，持续呼吸到交付 | 不遮挡订单商品槽 |
| 错误提示 | 450-700ms | 不遮挡商品区超过 300ms |
| 付款金币 | 800-1200ms | 从收银台飞向 HUD，不穿过当前订单槽中心 |
| Good / Perfect | 600-900ms | 当前顾客上方或收银台旁，避开 READY 气泡 |
| 低耐心警告 | 条件存在时循环脉冲 | 只作用于对应顾客和订单，不全屏闪 |
| 补货完成 | 600-900ms | 商品卡或经营前准备页库存区域，不在营业中弹大面板 |

### 隐藏 / 延后信息

以下信息不在营业页常驻显示：

| 信息 | 延后位置 | 原因 |
| --- | --- | --- |
| 商品进价、售价、利润率 | 经营前准备、结算、详情页 | 营业中不影响当前点击 |
| 全量库存报表 | 经营前准备 | 营业中只显示当前库存数字和售罄 |
| 需求预测、明日热门解释 | 经营前准备 | 营业页只显示必要角标 |
| 设备等级长文案 | 升级页 | 营业中只显示设备状态 |
| 历史收入曲线 | 结算或经营中心 | 与秒级服务无关 |
| 教学长说明 | 首次引导或暂停帮助 | 常驻会压低美术焦点 |
| 补货按钮和采购控制 | 经营前准备 | 营业中不允许打断服务节奏 |

## 层级优先级

这里的优先级指“画面冲突时优先保护谁”，不是简单的 z-index。HUD 有独立保护区，反馈不能盖住 HUD 关键数字；经营区内则以当前订单为最高焦点。

| 优先级 | 模块 | 必须保护的内容 | 可被弱化的内容 |
| ---: | --- | --- | --- |
| 1 | 当前顾客订单气泡 | 商品槽、缺失/已填、READY、点击范围 | 装饰光、气泡尾巴 |
| 2 | 商品区 | 商品图标、库存、需要/热食/售罄状态、点击范围 | 卡片装饰纹理 |
| 3 | 当前顾客 | 脸、表情、选中高亮、耐心 | 身体下缘、背景货架 |
| 4 | 微波炉 | heating/ready 状态、进度、可取出提示 | 炉体装饰、旁边小道具 |
| 5 | HUD | 时间、金币、暂停、主要数值 | 面板装饰、次级星级细节 |
| 6 | 收银台 | 付款数字、金币飞行起点 | 票据、小装饰、柜台纹理 |

补充规则：

- 当前顾客订单气泡进入 READY 后，优先级临时提升到全屏最高，仅低于系统暂停按钮。
- 微波炉处于 ready 状态时，优先级提升到与商品区同级；idle 时降到收银台同级。
- 收银台只在付款 1 秒内提升层级，结束后回到结果区。
- 非当前顾客的信息不能盖住当前顾客订单气泡，但其低耐心警告可以在自身气泡边缘脉冲。

## 竖屏保护区

### 390 x 844

390 宽度下比目标图更窄，不能按 750 等比硬缩。信息应保留纵向五段结构，次级装饰允许裁切或压缩。

| 区域 | 建议范围 | 不能被遮挡 |
| --- | --- | --- |
| HUD | y 0-68 | 时间数字、金币数字、暂停按钮；任何 toast 不得进入此区 |
| 顾客 / 订单 | y 72-320 | 当前订单气泡商品槽、READY、当前顾客脸、当前耐心条 |
| 工作台 | y 322-505 | 微波炉进度 / ready 灯、收银台付款数字出现点 |
| 商品区 | y 510-765 | 6 个商品卡的图标、库存角标、热食/需要/售罄标记 |
| 底部入口 | y 768-844 | 若保留底栏，图标可见；营业核心反馈不得落在这里 |

390 下的压缩规则：

- 当前顾客订单气泡不能低于 44px 高的可读商品槽。
- 当前顾客脸部不能被气泡、耐心条或付款反馈遮住。
- 商品卡图标最小可读高度建议不低于 52px，库存角标不低于 22px。
- 低耐心、错误、付款反馈优先贴边出现，不跨越订单气泡中心。
- 次要顾客可以缩小或局部裁切，但低耐心条和订单摘要不能完全消失。

### 750 x 1334

750 x 1334 与目标图比例接近，可以按目标图分区接入。

| 区域 | 建议范围 | 不能被遮挡 |
| --- | --- | --- |
| HUD | y 0-110 | 金币、时间、星级/连击、暂停 |
| 顾客 / 订单 | y 110-575 | 两位顾客的订单气泡、当前高亮、耐心条 |
| 工作台 | y 575-845 | 微波炉状态、收银台付款反馈 |
| 商品区 | y 845-1230 | 商品卡图标、库存、角标、点击范围 |
| 底部入口 | y 1230-1334 | 经营入口图标；营业中反馈不覆盖 |

750 下的保护规则：

- 当前订单气泡和商品区之间允许有飞行动效，但飞行拖尾不能遮挡商品库存数字。
- 收银金币可以从右侧穿过工作台飞到 HUD，但不能经过 READY 按钮中央。
- Good / Perfect 反馈优先出现在当前顾客肩部上方或收银台上方，避免盖住商品区。
- 次要顾客订单气泡可常驻，但视觉重量低于当前顾客。

## 状态反馈规则

### 错误

触发：

- 点击当前订单不需要的商品。
- 点击售罄商品。
- 订单未 READY 时尝试交付。
- 热食未完成时尝试填入订单。

出现：

- 错误源头局部红闪：商品卡、缺失槽、微波炉或订单气泡。
- 订单气泡左右 shake 2-3 次，单次幅度小于气泡宽度 3%。
- 可加一个红色感叹号或叉号，不使用大弹窗。

消失：

- 450-700ms 内自动淡出。
- 玩家下一次正确点击时立即清除错误态。
- 错误态不改变订单进度、不扣钱、不触发收银反馈。

### READY

触发：

- 当前订单所有必需商品槽填满。
- 热食完成并进入对应槽。

出现：

- 当前订单气泡描边变亮，READY 胶囊从气泡下方弹入。
- READY 同时配合勾选、发光和点击箭头，不能只靠文字。
- 当前顾客高亮保留，提示 READY 属于这位顾客。

消失：

- 点击 READY 气泡或当前顾客完成交付后，READY 切换为 DONE / 收款链路。
- 若顾客耐心耗尽，READY 直接消失并进入失败反馈。
- 交付成功后 300-500ms 内清掉旧槽位，切换到下一单。

### 低耐心

触发：

- 耐心低于 30% 进入警告。
- 耐心低于 15% 进入紧急。

出现：

- 对应顾客耐心条从黄到红。
- 订单气泡边框轻微脉冲，顾客表情变急，允许汗滴/怒脸图标。
- 只作用于对应顾客，不做全屏警报。

消失：

- 交付成功时立即清除，转开心/付款反馈。
- 顾客离开或失败时切换失败反馈，持续不超过 900ms。
- 若耐心恢复机制存在，回到 30% 以上后淡出警告。

### 付款

触发：

- READY 订单被成功交付。

出现：

- 收银台屏幕显示本单收入，例如 `+24`。
- 金币从收银台或顾客方向飞向 HUD 金币面板。
- HUD 金币数字 bump，数字变化和金币到达同步。

消失：

- 收银台数字 800-1200ms 后淡出或回到 idle。
- 金币粒子不保留在 HUD 上方。
- 付款反馈结束后收银台层级降低，不继续抢当前订单焦点。

### 补货

触发：

- 经营前准备页完成采购。
- 营业页出现售罄或低库存，需要把玩家引导到下一轮补货。

出现：

- 经营前准备页：商品卡库存数字上跳，绿色 `+N` 或补货小图标出现。
- 营业页：不出现补货按钮，只在商品卡显示低库存 / 售罄状态。
- 结算页：用一句短提示汇总“哪些商品需要补货”。

消失：

- 经营前准备页的 `+N` 反馈 600-900ms 后淡出。
- 营业页售罄状态持续到本轮结束或库存恢复。
- 补货建议进入结算或准备页，不在营业中弹出遮挡订单。

## 接入结论

- 经营页第一焦点是当前顾客订单气泡，READY 后它也是交付按钮。
- 商品区是第二焦点，因为它承接玩家最频繁点击。
- HUD 必须常驻清晰，但不参与经营区的动效抢层。
- 微波炉只在 heating/ready 时提升层级。
- 收银台是结果反馈，不是备餐托盘，也不是交付入口。
- 补货信息不应侵入营业主循环，营业中只显示库存压力，补货动作延后到准备页。
