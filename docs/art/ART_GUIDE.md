# 怪兽便利店美术规范

## 视觉目标

《怪兽便利店》采用明亮、友善、粗描边的扁平卡通风格。所有内容首先服务于手机竖屏中的可读性，其次才是细节表现。

正式风格参考：

- `assets/reference/style-exploration-flat-cartoon-v1.png`

## 核心规则

- 使用正面略带俯视的统一视角。
- 主轮廓使用深紫黑色，不使用纯黑。
- 角色、商品和设备使用圆润的大形状，避免细小结构。
- 单个资产最多使用 5 个主体颜色。
- 光源固定来自左上方。
- 高光使用简单色块，不使用写实反射。
- 阴影保持短、软、浅，不使用复杂环境阴影。
- 图片内禁止文字、品牌标识和写实包装信息。
- 资产缩小到游戏实际尺寸后，轮廓和用途仍应清晰。

## 怪兽模块规范

每只怪兽由可独立移动和替换的部件组成：

- `body`
- `belly`
- `eye`
- `mouth_neutral`
- `mouth_happy`
- `mouth_angry`
- `arm_left`
- `arm_right`
- `foot_left`
- `foot_right`
- `accessory_top`
- `accessory_side`

基础怪兽占屏幕高度约 22% 至 28%。眼睛和嘴巴必须足够大，确保表情在手机上清晰可见。

每只怪兽首版使用四种情绪：

- 等待：中性嘴型，偶尔眨眼。
- 开心：弯眼或放大眼睛，开心嘴型。
- 着急：耐心较低时显示紧张表情与汗滴。
- 生气：压低眼睛，生气嘴型，身体轻微变红。

## 商品规范

- 正方形构图，透明背景。
- 商品主体占画布约 80%。
- 包装图案使用简单图形，不出现可读文字。
- 同品类使用一致容器结构，通过颜色和中心图案区分。
- 在订单气泡中缩小显示时，仍能快速辨认。

建议首批商品：

- 薯片
- 柠檬汽水
- 饭团
- 草莓牛奶
- 怪兽布丁
- 星星糖果

## UI 规范

- UI 使用奶油白作为主要底色，深紫黑描边。
- 按钮保持清晰的主次层级，不使用复杂渐变。
- 数值、倒计时和连击反馈优先使用图标加数字。
- 游戏操作区不得被装饰元素遮挡。
- 竖屏设计基准为 `750 x 1334`。

## 动效语言

- 常规弹出：`0.18s`，先放大至 `1.08` 再回到 `1.0`。
- 正确操作：向上弹跳、绿色或黄色闪光、短促音效。
- 错误操作：左右摇晃、珊瑚红闪烁、低沉音效。
- 怪兽等待：身体缓慢呼吸，周期约 `1.8s`。
- 怪兽开心：向上跳跃一次，双臂抬起。
- 怪兽生气：快速抖动，头顶出现短时蒸汽粒子。
- 商品交付：沿弧线飞向收银台或顾客，不直接瞬移。

## 生成与验收流程

1. 新资产必须引用正式风格参考图。
2. 每批只生成同一类资产，例如一批只生成饮料。
3. 先生成少量样本，验收后再扩展。
4. 对最终资产统一执行裁切、尺寸、描边和色彩检查。
5. 在真机实际尺寸下检查，不以原图大图效果作为唯一判断。

## 禁止项

- 不混用 3D 黏土、像素、写实或厚涂风格。
- 不生成复杂毛发、透明玻璃或大量细碎装饰。
- 不为普通角色制作大规模逐帧动画。
- 不使用来源不明或无法确认商业授权的素材。
