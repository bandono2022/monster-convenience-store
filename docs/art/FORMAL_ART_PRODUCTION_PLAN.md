# 正式美术制作与接入计划

## 美术基准

正式竖屏玩法基准图：

- `assets/ui/mockups/gameplay-main-dual-customer-v2.png`

视觉方向：

- 珊瑚红、薄荷绿、奶油白、金黄和深紫为核心色板。
- 所有角色、商品、设备和 UI 使用一致的深紫粗描边。
- 角色使用少量赛璐璐明暗，不使用复杂纹理。
- 交互状态优先使用颜色、图标和动效表达。
- 双顾客、加工设备与商品区域必须保持清晰的视觉层级。

## 第一步：锁定垂直切片画面

状态：已完成

- 将单顾客旧概念改为双顾客调度布局。
- 明确顶部状态栏、双顾客区、加工区和商品区的层级。
- 明确订单需要、需加热、加热中和已交付的状态语言。

验收标准：

- 第一次看到画面的玩家能指出当前服务顾客。
- 玩家能识别哪些商品属于订单，以及哪个商品需要加热。
- 微波炉比普通装饰更醒目，但不遮挡订单和商品区。

## 第二步：制作第一批正式角色

状态：已完成

优先制作：

1. 珊瑚普通怪兽。
2. 黄色急性怪兽。

每只怪兽首批只制作四种状态：

- 等待。
- 开心。
- 着急。
- 生气。

接入方式：

- 角色身体保持固定尺寸和锚点。
- 表情变化尽量复用身体，只替换眼睛、嘴巴和动作层。
- 首版使用轻量帧动画或 Cocos 缩放、位移、旋转组合。

已完成资源：

- `assets/resources/game-art/characters/normal/`
- `assets/resources/game-art/characters/impatient/`

当前状态逻辑：

- 平时显示等待状态。
- 耐心低于 35% 时显示着急状态。
- 订单完成后短暂显示开心状态。
- 耐心耗尽后显示生气状态，再离开队列。

## 第三步：制作正式便利店场景与设备

状态：已完成第一版

制作内容：

- 便利店后景。
- 顾客柜台。
- 微波炉空闲、加热中、完成三个状态。
- 商品托盘和交付台。

拆分原则：

- 背景作为静态图降低性能成本。
- 微波炉门、内部光效、完成提示独立拆分。
- 设备状态使用可复用光效，不为每一帧绘制完整设备。

已完成资源：

- `assets/resources/game-art/environment/store-background.png`
- `assets/resources/game-art/equipment/microwave/idle.png`
- `assets/resources/game-art/equipment/microwave/heating.png`
- `assets/resources/game-art/equipment/microwave/ready.png`

当前接入效果：

- 便利店后景作为静态图层，顾客与操作区使用半透明色块保证可读性。
- 微波炉根据流程切换空闲、加热和完成三种正式美术状态。
- 加热状态保留倒计时、进度条和所属顾客提示。
- 完成状态使用金色边框与设备内部闪光引导玩家点击交付。

后续补充：

- 顾客柜台与商品托盘将在正式 UI 统一阶段一起制作，避免与商品卡片重复设计。

## 第四步：统一商品与 UI

状态：已完成第一轮结构重构

制作内容：

- 统一六种商品的画布尺寸、描边、阴影和视觉重心。
- 正式金币、时间、连击、暂停按钮。
- 订单气泡、耐心条、库存标签和状态徽章。
- 点击正确、点击错误、需要加工和完成交付图标。

验收标准：

- 商品缩小到订单图标尺寸后仍能区分。
- 所有状态颜色在手机屏幕上保持明显差异。
- UI 不依赖长文字解释。

第一轮已完成：

- 顾客卡片压缩高度并改用半透明底色，让便利店背景参与画面。
- 当前服务状态与顾客类型合并为单一标签，避免信息重叠。
- 新增独立加工台与柜台结构，固定微波炉和当前服务目标的位置。
- 商品状态徽章简化为“需要、加热、×库存”，减少文字拥挤。
- 顶部金币改为营业额表达，和营业目标保持一致。

下一轮仍需完成：

- 为营业额、时间和连击制作正式图标。
- 将商品卡片的文字徽章进一步替换为正式状态图标。
- 制作正式订单气泡、耐心条和商品托盘样式。

## 第五步：接入首轮正式动效

动效优先级：

1. 商品点击按压与飞向订单。
2. 微波炉加热、完成闪光和弹跳。
3. 顾客等待、开心和生气反馈。
4. 金币飞向顶部和连击反馈。
5. 顾客进店与离店。

性能原则：

- 常驻角色动画使用少帧循环。
- 爆发反馈优先使用 Cocos Tween 和粒子。
- 同屏粒子数量受控，适配微信小游戏低端设备。

## 第六步：试玩验收

- 使用微信开发者工具进行低端机性能测试。
- 首次接触游戏的玩家应在 30 秒内完成普通订单。
- 首次热饭团流程完成率目标高于 80%。
- 玩家点击商品后应在 0.2 秒内看到明确反馈。
- 美术接入后再调整难度，避免视觉不清造成伪难度。
